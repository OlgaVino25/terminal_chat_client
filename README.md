# Консольный чат-клиент

Этот проект – консольная утилита для взаимодействия с анонимным чатом.

**Программа позволяет:**

- **Читать** сообщения чата в реальном времени и сохранять их в файл с временными метками.
- **Отправлять** сообщения, авторизуясь по токену.
- **Регистрировать** нового пользователя и получать токен.

Код написан на Python с использованием `asyncio`, разбит на логические модули, устойчив к обрывам соединения и экранирует служебные символы (`\n`, `\r`), чтобы не нарушать протокол.

## Структура проекта

```text
├── config/               # Конфигурационные файлы
│ ├── read.ini            # Настройки для чтения чата (хост, порт)
│ ├── send.ini            # Настройки для отправки (хост, порт, токен)
│ └── reg.ini             # Настройки для регистрации (хост, порт)

│ ├── chat_history.log    # История переписки
│ └── token.txt           # Сохранённый токен (после регистрации)
├── src/                  # Исходный код
│ ├── init.py
│ ├── api.py              # Общие функции: connect, authorise, submit_message, sanitize_text
│ ├── client.py           # Скрипт чтения чата
│ ├── logger.py           # Класс ChatLogger (запись в файл + консоль)
│ ├── paths.py            # Абсолютные пути к папкам и файлам
│ ├── register.py         # Регистрация нового пользователя
│ └── sender.py           # Отправка одного сообщения
├── .gitignore
├── pyproject.toml
├── README.md
└── uv.lock
```

## Установка и запуск

**Требования**

- Python 3.13 или новее
- Установленный пакетный менеджер `uv` (или можно использовать `pip`)

**Установка**

1. Клонируйте репозиторий

2. Создайте виртуальное окружение и установите зависимости

```bash
uv venv
source .venv/bin/activate     # Linux / WSL
uv sync
```

**Запуск скриптов**

Все скрипты запускаются из корня проекта как модули Python:

```bash
python -m src.client                                         # чтение чата (порт 5000)
python -m src.sender "Ваше сообщение" --token <ваш_токен>    # отправка сообщения
python -m src.register --nickname "МойНик"                   # регистрация
```

## Подробное описание

**1. `client.py` – чтение чата**

Подключается к серверу, читает все входящие сообщения, выводит их в консоль и одновременно сохраняет в файл data/chat_history.log с временными метками вида [25.04.26 15:30].

**Особенности:**

- Устойчив к потере соединения: при ошибке выводит сообщение и завершается без `traceback`.
- Создаёт папку `data/`, если её нет.
- Поддерживает настройки через аргументы командной строки, переменные окружения или файл `config/read.ini`.

```bash
# Примеры
python -m src.client
python -m src.client --host 192.168.0.1 --port 5001 --history ~/my.log
```

**2. sender.py – отправка сообщения**

Авторизуется по токену и отправляет одно сообщение, после чего завершает работу.

Токен можно указать:

- через аргумент `--token`
- через переменную окружения `MINECHAT_TOKEN`
- через файл `config/send.ini`

Обязательный аргумент – текст сообщения (позиционный).

```bash
python -m src.sender "Привет, чат!" --token <ваш_токен>

# С переменными окружения

export MINECHAT_TOKEN=<ваш_токен>
export MINECHAT_SEND_PORT=<port>
python -m src.sender "Hello"
```

**3. register.py – регистрация нового пользователя**

Подключается к порту, отправляет пустой хэш, затем вводит ник (или берёт из аргумента `--nickname`). Полученный JSON с `account_hash` сохраняет в файл `data/token.txt`.

```bash
python -m src.register --nickname "Peaceful"

# или без ника – будет запрошен интерактивно

python -m src.register
```

После регистрации вы можете скопировать токен из `data/token.txt` и использовать его в `sender.py`.

## Настройка через конфигурационные файлы

Все три скрипта поддерживают файлы .ini (см. папку config/).

Тогда запуск `python -m src.sender "Сообщение"` автоматически подхватит эти настройки.

Приоритет: аргумент CLI > переменная окружения > файл конфигурации > значение по умолчанию.

## Экранирование служебных символов

Все строки (ник, текст сообщения, токен) перед отправкой проходят через функцию `sanitize_text()`, которая заменяет символы `\n` и `\r` на пробелы. Это предотвращает разрыв протокола, если пользователь случайно вставит перенос строки.

## Технические детали

- Асинхронное взаимодействие через `asyncio.open_connection`.
- Логирование (уровень `DEBUG` для `sender.py`, `INFO` для `register.py`).
- Использование `configargparse` для гибкой конфигурации.
- Отсутствие глобальных переменных, все зависимости передаются явно.
